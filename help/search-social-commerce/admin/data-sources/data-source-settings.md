# Inställningar för datakälla för [!DNL Google Analytics]

| Avsnitt | Parameter | Beskrivning |
| ---- | ---- | ---- |
| [!UICONTROL Connect to Google Analytics] | [!UICONTROL Google Analytics Account ID] | ID:t för det [!DNL Google Analytics]-konto som data hämtas från. All API-användning för att hämta data faktureras det angivna kontot. Kontot måste ge behörighet att läsa och analysera till den angivna e-postadressen.<br><br>Logga in på [!DNL Google Analytics] om du vill hitta ditt ID. Klicka på **[!DNL All accounts]** uppe till vänster för att öppna en lista över dina konton. ID:t för varje konto finns under kontonamnet. |
| | [!UICONTROL Google Analytics Login] | Ange den inloggnings-/e-postadress som ska användas för att komma åt data för den här datakällan. Inloggningen måste registreras för ett [!DNL Google]-konto och ha behörigheten Läs och analysera för [!DNL Google Analytics]-kontot. Se [instruktionerna om hur du tilldelar användarbehörigheter i  [!DNL Google Analytics]](https://support.google.com/analytics/answer/9305587).<br><br><b>Obs!</b> Om du ändrar lösenordet för det här inloggningskontot stängs alla öppna anslutningar till kontot. Om du vill återuppta synkroniseringen av data går du tillbaka till den här sidan och [återautentiserar](data-source-reauthenticate.md). |
| [!UICONTROL Account Details] | [!UICONTROL Google Analytics Account Name] | (Skrivskyddat, endast anslutna konton) Kontonamnet. |
| | [!UICONTROL Google Analytics Property] | Egenskapen (webbplats, mobilprogram eller enhet) som data ska samlas in från för kontot [!DNL Google Analytics].<br><br> Om du vill integrera mätvärden för flera egenskaper skapar du en separat datakälla för varje egenskap. |
| | [!UICONTROL Google Analytics View] | Vyn som innehåller den datauppsättning som du vill komma åt. Om egenskapen har flera vyer bör du dra den ofiltrerade vyn på den högsta nivån, som innehåller alla data.<br><br>Om du vill integrera mätvärden för flera vyer för samma egenskap skapar du en separat datakälla för varje vy. Se till att vyerna inte innehåller överlappande data. |
| | [!UICONTROL Google Analytics Dimension] | Den anpassade dimensionen [!DNL Google Analytics] som har fyllts i med värdet för frågesträngsparametern &quot;ef_id&quot; i Adobe Advertising. Om du inte ser rätt dimension i listan kontaktar du ditt [!DNL Google Analytics]-implementeringsteam.<br><br>ef_id används som primärnyckel för att skicka data från [!DNL Google Analytics] till Adobe Advertising. |
| [!UICONTROL Import Metrics] | [!UICONTROL Available Metrics] | Alla tillgängliga mått för den angivna [!DNL Google Analytics]-egenskapen och vyn som inte importeras för datakällan, ordnade efter kategori. Listan innehåller det importerade egna namnet och servernamnet (med början &quot;ga.&quot;) för varje mätvärde. Knappen [!UICONTROL Refresh] uppdaterar listan med nya mått i [!DNL Google Analytics].<br><br>Om du vill importera ett tillgängligt mått drar du det till avsnittet [!UICONTROL Selected Metrics].<br><br>Se [Tillgängliga [!DNL Google Analytics] Mätvärden i Advertising Cloud](data-source-ga-metrics.md).&quot; |
| | Valda mått | Alla mätvärden för den angivna [!DNL Google Analytics]-egenskapen och vyn som importeras för datakällan, samt valideringsstatus för varje mätvärde. Listan innehåller det importerade egna namnet och serverdelsnamnet (med början från `ga.`) för varje mätvärde. Varje datakälla kan innehålla fyra standardvärden för trafik som du inte kan ta bort ([!UICONTROL Pageviews], [!UICONTROL Sessions], [!UICONTROL Bounces] och [!UICONTROL Session Duration]) och upp till 16 ytterligare giltiga värden eller mått utan data. Du kan redigera måttlistan när som helst.<br><br>Om du vill importera ett mätresultat markerar du det i rutan [!UICONTROL Available Metrics] och drar det här.<br><br><b>Varning:</b> [!DNL Google Analytics] tillåter upp till 10 mätvärden i en enda datafeed. Var och en av dina datakällor i Search, Social och Commerce kan innehålla upp till två feeds med totalt 20 mätvärden, men om du använder en andra feed dubbleras dina API-anrop till [!DNL Google Analytics]. Om du har många mätvärden väljer du bara de mätvärden som du vill använda i optimeringsmål. Du kan visa dina kvoter för det här projektet i [den [!DNL Google API Console]](https://console.developers.google.com/apis/api/analytics-json.googleapis.com/quotas). Se mer om [kvoter och anropsgränser för API-begäranden till [!DNL Google Analytics]](https://developers.google.com/analytics/devguides/reporting/core/v4/limits-quotas).<br><br><b>Anteckningar:</b><br><ul><li>Det egna namnet importeras som visningsnamn för ett mått i vyn [!UICONTROL Admin] > [!UICONTROL Transactions Properties] och läggs till med <code>_ga:&lt;metrisk_tagg konfigurerad i fältet Metrisk tagg></code>&quot; (till exempel &quot;Pageviews_ga:UK&quot;). Du kan också redigera visningsnamnet för dina anpassade mål och anpassade mätvärden, men visningsnamnen för alla vanliga mätvärden skrivs över varje dag med de egna namnen i [!DNL Google Analytics], som läggs till med de angivna taggarna.</li><li>Sidvyer, sessioner, avhoppsfrekvens (beräknas som studsar/sessioner) och sessionens varaktighet räknas automatiskt in i algoritmerna för portföljbudgivning. Du kan lägga till andra mått manuellt till ett portföljmål.</li><li>När du tar bort ett mätvärde från datakällan behåller Adobe Advertising historikdata enligt den vanliga [datalagringspolicyn](/help/search-social-commerce/reports/data-used-for-reports.md).</li></ul> |
| Måtttagg | Måtttagg | En tagg som ska läggas till i varje markerat [!DNL Google Analytics]-mått i Adobe Advertising, föregånget av <code>_ga:</code>&quot; (till exempel &quot;Pageviews_ga:&lt;metric_tag>&quot;). Taggen kan bestå av 2-5 alfanumeriska tecken och måste vara unik för annonsören.<br><br>Med den här taggen kan du identifiera datakällan för varje mätvärde. Den här taggen är särskilt viktig när du konfigurerar flera datakällor eftersom varje datakälla innehåller några av samma måttnamn (inklusive [!UICONTROL Pageviews], [!UICONTROL Sessions], [!UICONTROL Bounces] och [!UICONTROL Session Duration] samt eventuellt andra mått). Om du lägger till olika metriska taggar förhindrar du att det finns flera metriska namn.<br><br>Om du till exempel skapar separata integreringar för din UK-egenskap och JP-egenskap kan du använda&quot;UK&quot; och&quot;JP&quot; som måtttaggar. Måtten [!UICONTROL Pageviews] för de två egenskaperna visas sedan i Adobe Advertising som&quot;Pageviews_ga:UK&quot; och&quot;Pageviews_ga:JP&quot;. |
